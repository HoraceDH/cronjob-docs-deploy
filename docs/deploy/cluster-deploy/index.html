      一、环境要求

      • OS：支持Linux、Windows、MacOS，下面以CentOS为例。
      • JDK：1.8+，官方用的是：1.8.0_391，64位。
      • MySQL：5.7+，官方用的是： 5.7.35-log Source distribution。

      二、下载地址

      • GitHub：https://github.com/horacedh/cron-job
      • GitEE：https://gitee.com/horacedh/cron-job

      三、初始化数据库

      从源码根目录下，找到mysql.sql文件，进入到MySQL命令行中，执行以下命令即可完成初始化，其中path替换为自己的路径：

      source /path/mysql.sql

      四、服务端部署

      1. 修改配置文件

      主要修改以下配置：

      • server.id：用于标识实例的唯一性，每个实例必须唯一。
      • server.port：服务端口，如果在同个机器部署，则需要修改端口。
      • manager.signKey：管理后台UI与调度器通信的签名Key，如需自定义，则需要与管理后台UI的配置保持一致，在cronjob-managerui/config/defaultSettings.ts中的signKey。
      • executor.signKey：执行器与调度器通信的签名Key，如需自定义，则需要与执行器的配置保持一致。

      详细的配置说明请参考：详细配置

      2. 依次启动调度器
      • 在每一个调度器的根目录下，依次执行以下命令：
      sh start.sh

      五、前端UI部署

      前端UI可以在分发包中找到build/cronjob-managerui目录，该目录就是打包后的前端文件，配置Nginx即可使用，需要与Nginx部署在同一台机器上。

      六、Nginx配置

      请参照如下配置文件，其余可按需调整配置：

      demo.cronjob.horace.cn.conf
      upstream demo.cronjob.horace.cn.server {
    # 配置多个调度器地址，用于负载均衡
    server 127.0.0.1:9527;
    server 172.16.108.40:9527;
}

server {
    listen  80;
    server_name demo.cronjob.horace.cn 172.16.62.4;
    error_log   logs/demo.cronjob.horace.cn.error.log   error;
    access_log  logs/demo.cronjob.horace.cn.access.log  main;

    location / {
        # 用于配合ant.design browserHistory 使用
        try_files   $uri $uri/index.html /index.html;
        root    /data/software/cronjob/cronjob-managerui;
        index   index.html;
    }

    location /manager-api {
        proxy_pass  http://demo.cronjob.horace.cn.server;
        proxy_set_header    X-Forwarded-Proto $scheme;
        proxy_set_header    Host              $http_host;
        proxy_set_header    X-Real-IP         $remote_addr;
    }

    location /openapi {
        proxy_pass  http://demo.cronjob.horace.cn.server;
        proxy_set_header    X-Forwarded-Proto $scheme;
        proxy_set_header    Host              $http_host;
        proxy_set_header    X-Real-IP         $remote_addr;
    }
}
